# gitflow
---
## gitflow란
---
Git Flow는 Git에서 사용하는 브랜치 관리 방법(규칙),쉽게 말하면 여러 사람이 협업할 때 작업을 정리하는 방법으로 프로젝트를 여러 브랜치로 나눠서 관리

---
## gitflow의 주요 브랜치
---
1.main (또는 master)
완성된 최종 코드
실제 서비스에 사용되는 안정 버전

2.develop
개발 중인 코드
여러 기능이 모이는 곳

3.feature 브랜치
새 기능 개발할 때 사용
develop에서 만들어서 작업 후 다시 합침

4️.release 브랜치
배포 준비 단계
버그 수정, 테스트 진행

5️.hotfix 브랜치
긴급 버그 수정
main에서 바로 수정 후 다시 합침

---

## gitflow의 장단점
<장점>
1.체계적인 관리
브랜치가 역할별로 나뉘어 있어서 작업 흐름이 명확함

2️.협업에 좋음
여러 사람이 동시에 작업해도 충돌을 줄일 수 있음

3️.안정적인 배포
main은 항상 안정된 상태 유지, 오류가 적음

<단점>
1️.구조가 복잡함
브랜치 종류가 많아서 처음 배우기 어려움

2️. 작업 속도 느림
merge 과정이 많아서 개발 속도가 느려질 수 있음

3️.작은 프로젝트에 비효율
간단한 프로젝트에는 너무 과한 방식
